# ADR 003: フェーズ2（次期開発）における技術要素の優先順位とアーキテクチャ方針

- **Date:** 2026-05-26
- **Status:** Accepted
- **Context (背景):**
  フェーズ1における初期PoC（FastAPIからのダミーデータ取得とFlutterでのダッシュボード表示）が成功した。
  本プロジェクトを「世界標準のフルスタック・ポートフォリオ」として完成させるにあたり、フェーズ2で追加すべき技術要素（無限スクロール、グラフUI、CI/CD、gRPC/GraphQL等）の優先順位を決定する必要があった。
  意思決定にあたり、2024-2025年のグローバルなモバイルアプリ・アーキテクチャのベストプラクティスおよび、テック業界の採用担当者が評価する基準についてファクトチェックを実施した。

- **Decision (決定事項):**
  ファクトチェックのエビデンスに基づき、以下の優先順位で実装を進めることを決定した。

  **優先度1: カーソルベースのページネーション（無限スクロール）の実装**
  - **理由:** モダンなモバイルフィードUIにおけるデータの追加読み込みは、従来のオフセットベースではなく「Cursor-Based Pagination」がデファクトスタンダードである。これを実装することで、バックエンド（FastAPI）の堅牢なAPI設計能力と、フロントエンド（Flutter/Riverpod）の高度な非同期状態管理能力の双方を同時に証明する。

  **優先度2: CI/CDパイプラインの自動化と Live Demo の公開**
  - **理由:** 採用担当者の84%が「実際に動作するアプリケーション」を求めているというデータに基づき、ローカルでの実装完了後、速やかに GitHub Actions 等を用いて Cloud Run への自動デプロイ環境を構築する。これにより、DevOps / IaC のスキルセットを証明する。

  **優先度3: UI/UXの強化 (Charts) および APIプロトコルの高度化 (gRPC / GraphQL)**
  - **理由:** グラフの追加や、エンタープライズ標準である gRPC / GraphQL への移行は非常に価値が高いが、基礎的な機能とインフラ基盤（Live Demo）が完成した後の「フェーズ3」として着手し、ポートフォリオの完成度を段階的に高めるアプローチをとる。

- **Consequences (結果と影響):**
  - **ポジティブな影響:**
    - 開発リソース（時間）を、最も市場価値（ROI）の高いタスクに集中投資できる。
    - このADR自体が、「客観的なファクト（業界動向）に基づいて技術選定と優先順位付けができるシニアエンジニア」であることの強力な証明となる。
  - **今後の課題:**
    - バックエンドのAPI仕様を変更（オフセットからカーソルベースへ）するため、フロントエンドとバックエンドのコントラクト（I/F）を壊さないよう、Schema-Driven のアプローチを維持して実装する必要がある。
